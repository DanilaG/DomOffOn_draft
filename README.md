#  ДомOffOn (MVP Draft)

Драфт MVP приложение для удалённого управления домофоном.

## Запуск проекта

1. Клонирование проекта.
2. Установит pod-s:
```pod install```.
3. Запуск workspace.

## Развитие проекта с точки зрения приложения

### Выпуск MVP

1. Реализовать модуль авторизации с возможностью входа.
2. Реализовать модуль OTP.
3. Реализовать модуль главного экрана с кнопкой открытия домофона.
4. Реализовать открытие домофона через команды Siri.
5. Внедрение метрики. Предварительно Яндекс.Метрика.
6. Выпуск на TestFlight.
7. Тестирование 2-х UX гипотез и одной бизнес. 

Переход к следующим шагам только при подтверждении всех 3х гипотез. 

### Подготовка приложения к публикации

1. Разбитие приложения на изолированные pod модули. Иначе высок риск перемешивания кода.
2. Покрытие имеющийся логики unit тестами. Предварительно, для unit тестов использовать Quick.
3. Чистка файлов.
4. Исправление варнингов.
5. Добавление показа ошибок.
6. Настройка CI/CD.
7. Приведение приложения в соответствие с требованиями публикации Apple. 
8. Исправление замечаний со стороны UI/UX, копирайтеров.
9. Добавление UI тестов. Предварительно SnapshotTesting.

При достижение более 300 активных пользователей и не менее 5% платящих пользователей перейти к следующему этапу.

### Развитие приложения

1. Уход на собственный бэк.
2. Добавление генерации кода.

## Правила ведения проекта

### Написание кода

1. Следовать правилам линтера.

### Документирование кода

1. Документирование кода производится на русском (ruenglish) языке.
2. Документированием покрываются все не приватные методы, поля, классы, структуры, акторы, протоколы...
3. Поля и методы порождённые покрытым документацией протоколом освобождаются от документирования.

### Добавление фитчи

1. Каждая фитча находится в своём отдельном модуле.
2. Модуль фитчи не зависит от модуля другой фитчи.
3. Навигация между фитчами производится через координатор.
4. Для навигации внутри модуля разрешено использовать внутренний роутер, в том случае, если навигация осуществляется между непереиспользуемыми (маловероятно повторно используемыми) компонентами.
